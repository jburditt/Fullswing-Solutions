HTML Structure

Vittoria is built on it's own responsive grid system. The syntax is simple and it's effective cross browser. You can easily customize all parts of your site or build a new ones.

Basic HTML template

<!DOCTYPE HTML>
<html lang="en-US">
<head>
    <meta charset="UTF-8">
    <title>Vittoria</title>
    <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
    <!-- Insert here all CSS and JS links from a sample template -->
</head>
<body class="l-body">
<div class="l-background"></div>
<!-- CANVAS -->
<div class="l-canvas type_boxed col_cont">
    <div class="l-canvas-h">
        <!-- HEADER -->
        <div class="l-header type_normal">
            <div class="l-header-h">
            <!-- Header sections -->
            </div>
        </div>
        <!-- /HEADER -->
        <!-- MAIN -->
        <div class="l-main">
            <div class="l-main-h">
                <div class="l-submain for_pagehead">
                    <div class="l-submain-h i-cf">
                    <!-- Page head content (this section is optional) -->
                    </div>
                </div>
                <div class="l-submain">
                    <div class="l-submain-h">
                    <!-- Page content -->
                    </div>
                </div>
            </div>
        </div>
        <!-- /MAIN -->

    </div>
</div>
<!-- /CANVAS -->
<!-- FOOTER -->
<div class="l-footer type_normal">
    <div class="l-footer-h">
    <!-- Footer sections -->
    </div>
</div>
<!-- /FOOTER -->
</body>
</html

Key features are listed below

1. Easily switch between boxed and wide layout

Boxed or wide layout is switched with modifier of canvas div. Find the div with class l-canvas on the page and add class type_boxed or type_wide class in order to switch the layout.

Boxed (.type_boxed):

<!-- CANVAS -->
<div class="l-canvas type_boxed col_cont">

Wide (.type_wide):

<!-- CANVAS -->
<div class="l-canvas type_wide col_cont">

Do not use both type_boxed and type_wide modifiers together!

2. Responsive grid

A responsive grid row structure is shown below. Each row should be wrapped in .g-cols container. You can use human-readable class names like .one-quarter or .two-thirds.

<div class="g-cols">
    <div class="one-third">
        <h3>One Third</h3>
        <p>Lorem ipsum dolor sit amet...</p>
    </div>
    <div class="one-third">
        <h3>One Third</h3>
        <p>At delectus doloremque dolores explicabo...</p>
    </div>
    <div class="one-third">
        <h3>One Third</h3>
        <p>Accusamus et hic inventore iure iusto modi...</p>
    </div>
</div>

Animated Elements

You can animate appearance of any element on any page. Just add "animate_*" to class of your element in HTML syntax:

<div class="element animate_afc">...</div>
<div class="element animate_rfl">...</div>
...

Full list of animation variants:

  • animate_afc — Appear From Center
  • animate_afl — Appear From Left
  • animate_afr — Appear From Right
  • animate_aft — Appear From Top
  • animate_afb — Appear From Bottom
  • animate_wfc — Width From Center
  • animate_hfc — Height From Center
  • animate_rfc — Rotate From Center
  • animate_rfl — Rotate From Left
  • animate_rfr — Rotate From Right

For elements in one row you can use delay of appearance by adding "d*":

<div class="element_1 animate_afc">...</div>
<div class="element_2 animate_afc d1">...</div>
<div class="element_3 animate_afc d2">...</div>
<div class="element_4 animate_afc d3">...</div>

Delays:

  • d1 — 0.2 seconds
  • d2 — 0.4 seconds
  • d3 — 0.6 seconds
  • d4 — 0.8 seconds
  • d5 — 1 second

Important: many elements with animation on one page can put a heavy strain on user computer's CPU and browser can work very slow.

Twitter

Our Twitter widget is based on jquery.tweet.js-mod plugin by Stan Scates. The  widget is simple to use and configure.

First, you need to have a twitter App for your usage in order to obtain OAuth credentials, see https://dev.twitter.com/apps for help.

After obtaining OAuth credentials, put them to inc/twiter/index.php. Find and edit following code:

/* Configure your twitter API access params below */
// Your Twitter App Consumer Key
$consumer_key = 'YOUR_CONSUMER_KEY';

// Your Twitter App Consumer Secret
$consumer_secret = 'YOUR_CONSUMER_SECRET';

// Your Twitter App Access Token
$user_token = 'YOUR_ACCESS_TOKEN';

// Your Twitter App Access Token Secret
$user_secret = 'YOUR_ACCESS_TOKEN_SECRET';

Then, go to js/plugins.js and find this code:

jQuery(".w-twitter-tweets").tweet({
    join_text: "auto",
    modpath: "inc/twitter/",
    username: [""],
    avatar_size: 48,
    count: 2,
    template: "<i class='icon-twitter'></i><span>{text}</span> <small>{time}</small>",
    loading_text: "loading tweets..."
});

You can alter the amount of loaded tweets here, by setting needed number:

    count: 3,

You may also specify twitter username to fetch tweets from. By default it is blank, because it will fetch tweets from user, who owns the Twitter App, that you configured at inc/twitter/index.php. So, for example, if you put "envato" here:

    username: [""],

The widget will fetch tweets for envato user, but not for owner of the Twitter App. Pay attention to the syntax of this line. Do not erase the square brackets.

Next, if you want to place Twitter widget in a particular place different from footer or sidebar (as it is at sample pages), insert this code (you can find same code at sample pages) at the place where you want it to be:

<div class="w-twitter">
    <div class="w-twitter-h">
        <h3 class="w-twitter-title">Recent tweets</h3>
        <span class="w-twitter-tweets"></span>
    </div>
</div>

For further documentation about jquery.tweet plugin see it's project at GitHub  https://github.com/StanScates/Tweet.js-Mod

Migration Guide

Migrating from 1.0 to 1.1

  1. Renamed elements classes: 
    w-infobox => w-iconbox
w-intro => w-actionbox
  2. Changed HTML syntax in social links.
    Before (v 1.0): 
    <a class="w-socials-item-link" ... >***</a>
    After (v 1.1): 
    <a class="w-socials-item-link" ... ><i class="iconsocial-***"></i></a>

 

Migrating from 1.1 to 1.2

  1. Renamed elements classes: 
    w-side => widget
  2. Removed all "w-html" elements.
    Before (v 1.1): 
    ...
<div class="w-html g-html">
    <h3>Your Title</h3>
    <p>Your content</p>
</div>
...
    After (v 1.2): 
    ...
<h3>Your Title</h3>
<p>Your content</p>
...
  3. Changed HTML syntax in all "l-submain-h" elements.
    Before (v 1.1): 
    <div class="l-submain-h">

<div class="l-submain-h i-cf">
    After (v 1.2): 
    <div class="l-submain-h g-html">

<div class="l-submain-h g-html i-cf">
  4. Changed HTML syntax in social links.
    Before (v 1.1): 
    <ul class="w-socials-list">
    <li class="w-socials-item ...">
        ...
    </li>
    ...
</ul>
    After (v 1.2): 
    <div class="w-socials-list">
    <div class="w-socials-item ...">
        ...
    </div>
    ...
</div>

 

Migrating from 1.2 to 1.3

Deleted block with fullwidth-image on home-3.html:

<div class="fullwidth-image" style="background-image: url(img/slides/slide4.jpg);"></div>

Now all background images set to .l-background block.
